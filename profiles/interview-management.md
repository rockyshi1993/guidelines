# interview-management 项目规范（AI 助手专用）

> **核心目标**: 通过 AI 分析候选人简历，生成标准化评估报告和面试问题，帮助招聘到最合适的人才
> **版本**: v1.0  
> **更新日期**: 2025-11-03

---

## 📋 项目信息

- **项目类型**: 文档型项目（面试管理系统）
- **主要用户**: 技术面试官
- **核心技术**: Markdown 文档 + PowerShell 脚本
- **AI 角色**: 简历分析师 + 面试问题生成器
- **隐私级别**: 私有仓库（包含候选人敏感信息）

---

## 🎯 AI 工作流程

### 典型任务场景

```yaml
用户输入:
  - candidates/[目录]/简历原文.md（候选人简历）
  - jobs/[岗位].md（岗位招聘条件）

AI 输出:
  - 快速概览.md（30秒判断 + 匹配度评分）
  - 简历分析.md（优势、疑点、红旗）
  - 题目清单.md（60-90分钟面试问题）

核心原则:
  - 客观量化评估（0-100% 匹配度）
  - 基于 JD 的多维度分析
  - 生成针对性验证问题
  - 识别候选人红旗
```

#### 评分规则

**技术栈匹配度计算**:
```python
匹配度 = (匹配技能数量 / JD要求技能数量) × 100%

评级标准:
  90-100%: ⭐⭐⭐⭐⭐ 完全匹配
  80-89%:  ⭐⭐⭐⭐   高度匹配
  70-79%:  ⭐⭐⭐     基本匹配
  60-69%:  ⭐⭐       部分匹配
  <60%:    ⭐         匹配度低
```

**工作稳定性评估**:
```python
平均工作时长计算:
  总工作年限 / 公司数量 = 平均年限

评分标准:
  ≥3年:   100% ✅ 稳定
  2-3年:  75%  ⚠️ 可接受
  1-2年:  50%  ⚠️ 需关注
  <1年:   20%  🚩 严重红旗
```

**综合评分算法**:
```python
技术能力得分 = 加权总分（基于 JD 各维度权重）
稳定性得分 = 工作稳定性评分

综合评分 = 技术能力得分 × 70% + 稳定性得分 × 30%
```

#### 必须识别的红旗

**严重红旗（必须标注 🚩）**:
- ❌ 频繁跳槽（平均 < 1 年）
- ❌ 简历疑似造假（时间线矛盾、技能夸大）
- ❌ 核心技能与 JD 完全不匹配
- ❌ 工作经历存在大段空白期（未说明）

**一般红旗（需要关注 ⚠️）**:
- ⚠️ 项目描述过于笼统（缺少具体数据）
- ⚠️ 技术深度不足（只列技能名称，无实际应用）
- ⚠️ 职业发展方向不清晰
- ⚠️ 跨领域跳转过于频繁

```
你是一位经验丰富的技术面试官，请根据以下材料进行候选人评估：

# 输入材料

## 候选人简历
[粘贴 candidates/[目录]/简历原文.md 完整内容]

## 岗位要求
[粘贴 jobs/[岗位].md 完整内容]

# 分析任务

请生成以下三个完整的 Markdown 文档：

## 1. 快速概览.md
要求：
- 基本信息表格（姓名、联系方式、学历、经验、地点）
- 30秒快速判断（技术栈匹配、项目经验、工作稳定性、技术广度）
- 核心亮点（3-5个，每个都要说明与 JD 的关联）
- 核心风险（工作稳定性、技能短板等）
- 基于JD的匹配度评估表格（严格按照 JD 中的"评分权重"）
- 综合评分（技术能力 × 70% + 稳定性 × 30%）

评分规则：
- 每个维度给出 0-100% 的量化评分
- 必须基于 JD 中的权重计算加权总分
- 必须识别所有红旗（严重和一般）
- 根据级别标准判断是否推荐面试

## 2. 简历分析.md
要求：
- 优势亮点（分类：技术栈、项目经验、其他）
- 疑点和需要深挖（每个疑点都要提供面试策略）
- 红旗预警（严重红旗、一般红旗）
- 综合评估和建议

疑点识别重点：
- 项目规模不明确（无 QPS、数据量）
- 技术深度不足（只列名称无应用）
- 职责描述笼统（无具体产出）
- 时间线矛盾

## 3. 题目清单.md
要求：
- Part 1: 开场破冰（5-10分钟，2题）
- Part 2: 技术基础（20-30分钟，5-8题，根据级别调整难度）
- Part 3: 项目深挖（20-30分钟，完全基于简历定制）
- Part 4: 系统设计（15-20分钟，1-2题，与 JD 相关）
- Part 5: 行为面试（5-10分钟，3题）
- Part 6: 反问环节（5-10分钟）

问题设计原则：
- 每个问题必须包含：问题、考察点、评分标准
- 必须针对简历中的项目和技术定制
- 必须包含验证疑点的问题
- 必须标注红旗信号
- 难度分布：基础 40% + 进阶 40% + 深度 20%（根据级别调整）

# 输出格式

请严格按照以下参考格式输出：
- 参考 candidates/谢小东/ 目录下的文件格式
- 使用 Markdown 表格展示数据
- 使用 Emoji 标注状态（✅⚠️❌🚩🌟）
- 使用星级（⭐）标注匹配度

# 质量标准

- 必须量化评估（不能只说"好"或"不好"）
- 必须基于 JD 权重计算（不能主观打分）
- 必须识别所有红旗
- 必须为每个疑点提供验证策略
- 必须生成针对性问题（不能用通用模板）

# 注意事项

- 客观公正，不要有偏见
- 如果信息不足，明确标注"待补充"
- 如果发现矛盾，明确指出
- 如果有严重问题，必须突出警示
```

## ❓ FAQ

**Q: AI 分析需要多长时间？**  
A: 通常 2-5 分钟，取决于简历长度和 AI 响应速度。

**Q: AI 评分准确吗？**  
A: 作为初步筛选准确，但必须人工审核。建议 ±10% 误差范围。

**Q: 可以完全依赖 AI 吗？**  
A: 不可以。AI 用于提高效率，最终决策必须由人做出。

**Q: 如何提高 AI 分析准确性？**  
A: 1) 提供完整的简历和 JD  2) 使用详细的提示词  3) 人工审核调整

**Q: 哪个 AI 效果最好？**  
A: Claude 和 GPT-4 效果较好，GitHub Copilot 也不错。建议多试几个。
